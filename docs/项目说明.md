# 项目说明

## 我为你创建了什么

### 📁 项目结构

我已经为你创建了一个完整的项目骨架，包括：

1. **核心模块接口定义**（`src/core/`）
   - `llm/base.py` - LLM 基础接口
   - `embedding/base.py` - Embedding 基础接口
   - `vectorstore/base.py` - 向量存储基础接口
   - `chain/rag_chain.py` - RAG Chain 框架

2. **知识库管理**（`src/knowledge_base/`）
   - `kb_manager.py` - 知识库管理器框架

3. **Agent 模块**（`src/agent/`）
   - `state.py` - Agent 状态定义框架
   - `nodes.py` - Agent 节点框架
   - `graph.py` - LangGraph 图构建框架

4. **文档和指南**（`docs/`）
   - `学习路径.md` - 详细的学习计划
   - `架构设计.md` - 项目架构说明
   - `开发指南.md` - 开发技巧和提示
   - `快速开始.md` - 快速上手指南

5. **开发工具**
   - `TODO.md` - 分阶段的详细任务列表
   - `examples/basic_rag_example.py` - 使用示例
   - `requirements.txt` - 依赖列表
   - `.gitignore` - Git 忽略文件

### 🎯 设计原则

我采用了"**指导式开发**"的方式：

✅ **我提供的**：
- 清晰的架构设计
- 接口定义和 TODO 注释
- 详细的开发指南和学习路径
- 分阶段的 TODO 列表
- 关键代码框架和提示

❌ **我不提供的**：
- 完整的实现代码（核心逻辑需要你自己写）
- 直接可运行的代码（需要你先实现接口）

### 📝 关键文件说明

#### 1. `src/core/llm/base.py`
- 定义了 `BaseLLM` 接口
- 包含 `generate()` 和 `stream_generate()` 方法
- 有详细的 TODO 注释指导你实现

#### 2. `src/core/chain/rag_chain.py`
- **这是项目的核心**
- 包含 RAG 的完整流程框架
- 你需要实现检索、组装、生成三个步骤

#### 3. `src/agent/graph.py`
- LangGraph 的学习重点
- 展示了如何构建 Agent 图
- 需要你理解 State、Node、Edge 的概念

#### 4. `TODO.md`
- **最重要的文件**
- 包含4个阶段的详细任务
- 每个阶段都有检查点
- 按照这个列表逐步实现

## 接下来你应该做什么

### 第一步：阅读文档（30分钟）

1. 阅读 `README.md` 了解项目概况
2. 阅读 `docs/学习路径.md` 了解学习计划
3. 阅读 `docs/架构设计.md` 理解项目架构
4. 阅读 `docs/快速开始.md` 了解如何开始

### 第二步：环境准备（15分钟）

```bash
# 创建虚拟环境
python -m venv venv
venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
# 复制 .env.example 为 .env，填入你的 API Key
```

### 第三步：开始实现（按照 TODO.md）

**阶段一：基础 RAG（1-2周）**

1. 打开 `src/core/llm/base.py`
2. 阅读 TODO 注释
3. 参考 LangChain 文档实现 `OpenAILLM`
4. 测试你的实现
5. 重复上述步骤实现其他模块

**重要提示**：
- 不要跳过任何步骤
- 每实现一个模块就测试
- 遇到问题先查文档，再搜索，最后提问

### 第四步：持续学习

- 完成一个阶段后，回顾学到了什么
- 记录遇到的问题和解决方案
- 准备进入下一阶段

## 学习建议

### ✅ 推荐的学习方式

1. **理解优先**
   - 先理解概念，再写代码
   - 阅读 LangChain 官方文档
   - 查看示例代码

2. **逐步实现**
   - 按照 TODO 列表逐步完成
   - 每个模块独立测试
   - 不要急于完成所有功能

3. **主动思考**
   - 思考为什么这样设计
   - 尝试改进和优化
   - 记录学习心得

4. **实践验证**
   - 用真实文档测试
   - 验证答案质量
   - 优化参数和 prompt

### ❌ 避免的做法

1. **不要直接复制代码**
   - 即使找到示例，也要理解后自己实现
   - 复制粘贴无法提升能力

2. **不要跳过基础**
   - 每个阶段都很重要
   - 基础不牢，后面会困难

3. **不要孤立学习**
   - 理解模块之间的关系
   - 理解整体架构
